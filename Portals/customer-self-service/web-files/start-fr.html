Comment démarrer

Si vous travaillez derrière un proxy, vous pouvez vous référer au document guide de dépannage proxy.

Je veux utiliser la BOEW sur mon site Web

  1. Téléchargez les fichiers de production de la dernière version stable d'un des thèmes de la page de téléchargements.
  2. Assurez-vous que le serveur est configuré correctement.
  3. Suivez les instructions de mise en œuvre dans la documentation du thème qui a été téléchargé.
  4. Révisez le guide de style (anglais seulement) pour de l'information comment intégrer la BOEW à votre site.

Je veux personaliser la BOEW et créer un nouveau thème BOEW

  1. Installez NodeJS pour votre plateforme.
  2. Créer un répertoire pour votre projet.
  3. Installez Grunt CLI, Yeoman, and Bower packages globally by running npm install -g grunt-cli yo bower from the command line console (Command Prompt on Windows, Terminal on OS X, or other shell environment on Linux or Unix).
  4. Installez le générateur de thème BOEW en exécutant npm install -g generator-wet-boew-theme.
  5. Exécuter yo wet-boew-theme à partir de l'invite de commande, et répondez aux questions à l'invite pour configurer votre projet de thème.

Je veux signaler une anomalie ou demander une question à propos de la BOEW

  1. Créer un compte GitHub.
  2. Poser une question relative au guide de style (anglais seulement) en signalant une anomalie sur GitHub.
  3. Signaler une anomalie, demander une question ou demander une fonctionalité relative à la BOEW en suivant le guide de signalement.

Je veux corriger ou améliorer la BOEW en tant que développeur

  1. Installer NodeJS pour votre plateforme.
  2. Créer un compte GitHub.
  3. Suivre les guides GitHub à propos de configurer Git.
  4. Bifurquer le dépôt WET-BOEW en suivant le guide de bifurcation GitHub. Utilisez l'URL en amont de https://github.com/wet-boew/wet-boew.git à l'étape 3.
  5. Exécuter ./script/setup du répertoire wet-boew dans votre console de ligne de commande.
  6. Exécuter grunt pour bâtir le projet. Vous pouvez exécuter grunt --help pour voir les descriptions de cible de bâtissage.
  7. Les derniers fichiers seront alors compilés dans le répertoire dist/.
  8. Pour contribuer de nouveau, suivez les instructions au comment créer une demande de retrait.
